# PR 작성 지침

1. PR에는 변경 사항과 그 이유를 요약한 내용이 들어가야 합니다.
2. 종료 조건이 완료됐다는 사실을 증명할 내용이 들어가야 합니다.
3. PR의 크기는 LOC 500줄 이내로 제한합니다. 만약 크기가 더 커지면 여러 PR로 나누어 올립니다.
   1. 다음은 예외로 합니다. 자동 생성 문서, 계획 문서.
4. PR은 반드시 논리적인 작업의 한 단위로 묶여야 합니다.
5. 가능하면 CLI 도구를 사용합니다.

---

## 커밋 메시지 컨벤션 (Commit Message Convention)

커밋 시 다음 형식을 준수합니다:

```
type : subject

body

footer
```

- **type**: 변경 유형 (예: feat, fix, docs, style, refactor, test, chore)
- **subject**: 변경 사항에 대한 짧고 명령형의 요약
- **body**: 변경 사항의 상세 설명 (동기, 이전 동작과의 비교)
- **footer**: 이슈 참조 또는 외부 링크 (선택 사항 또는 필요 시)
