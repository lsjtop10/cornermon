# 구현 주의사항

- 새 워크트리 생성 여부 확인: 사용자에게 새 워크트리에서 작업할지 확인을 받으세요.

- 검증 중심 완료: 작업이 끝났다는 것은 구현이 끝난 것이 아니라 Plan 문서에 있는 검증까지 완료되었음을 의미합니다.

- 도메인 일관성: 구현 시 반드시 개발 가이드라인을 준수하며, 코드의 명명 규칙은 domain-model.md에 정의된 유비쿼터스 언어와 1:1로 일치해야 합니다.

- 실시간 Plan 업데이트: Plan 문서를 실시간으로 업데이트하여 작업 현황과 미해결 과제를 관리하세요.

- 논리적 최소 커밋: 하나의 커밋은 하나의 논리적 작업 단위입니다. 되도록 loc 300줄을 넘지 않도록 쪼개서 작업하세요.

- API 스키마 우선: Go-Flutter 간 통신 변경 시 API 스키마 정의 파일 수정 및 코드 생성을 최우선으로 진행합니다.

- 작업 범위 준수: 백엔드 issue는 백엔드 코드만 프론트엔드 issue는 프론트엔드만 수정합니다.


# 테스트 주의사항

- 테스트 이름 규칙: ShoudxxxWhenyyy 이름을 따릅니다.
- 테스트 코드는 arrange-act-assert 프레임워크르르 따릅니다. 주석에도 명시합니다.
