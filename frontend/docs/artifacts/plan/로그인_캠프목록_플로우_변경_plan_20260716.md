# 로그인 → 캠프 목록 플로우 변경 (관리자 앱)

> 상태: **계획만 작성, 구현 미착수**. 사용자 승인 후 구현 시작.
> 관련 문서 업데이트 완료: `docs/front/screen-spec-admin.md`(A0 후속 분기, A0-c 상태), `docs/front/scenarios.md`(Feature 2-d, 2-g).

## 1. 배경 및 유즈케이스

| 우선순위 | 유즈케이스 | 설명 | 용도 |
| --- | --- | --- | --- |
| **P0** | UC-1: 로그인 후 캠프 목록 고정 진입 | 캠프 개수(0개 포함)와 무관하게 로그인 성공 시 항상 캠프 목록(A0-c, `/camps`)으로 이동한다. 초기 설정 마법사(A0-b, `/setup-wizard`)로의 자동 분기를 제거한다. | **프로덕션 핵심 로직** |
| P1 | UC-2: 빈 캠프 목록의 empty state | 캠프가 0개일 때 캠프 목록 화면이 "아직 캠프가 없습니다" 안내와 "+ 새 캠프 시작" CTA를 보여준다. | 이미 `camp_list_screen.dart`에 구현됨(확인 필요, §3) |

**변경 이유**: 로그인 직후 목적지가 캠프 유무에 따라 갈리면 내비게이션을 예측하기 어렵다. "캠프 목록"을 유일한 로그인 후 진입점으로 통일하고, 캠프 생성은 항상 그 화면의 명시적 버튼을 통해서만 시작하게 한다.

## 2. 현재 코드 상태 (구현 전 조사 결과)

`frontend/lib/admin/router/admin_router.dart:100-124`의 `_redirect` 함수:

```dart
String? _redirect(Ref ref, String location) {
  if (ref.read(adminSessionProvider) is AdminSessionUnauthenticated) {
    return location == '/login' ? null : '/login';
  }
  if (location == '/login') {
    return ref
        .read(campListProvider)
        .whenOrNull(
          data: (camps) => camps.isEmpty ? '/setup-wizard' : '/camps',
        );
  }
  // ...
}
```

`/login`에서의 리다이렉트가 `campListProvider`의 캠프 개수를 보고 `/setup-wizard` 또는 `/camps`로 분기한다 — 이 분기가 제거 대상이다.

`frontend/lib/admin/features/camp_list/camp_list_screen.dart`에는 이미 empty state와 "+ 새 캠프 시작"(`context.go('/setup-wizard')`) 버튼이 존재하는 것으로 확인됨(line 29, 42, 50, 82) — UC-2는 별도 구현 없이 이미 충족될 가능성이 높다. 구현 착수 시 실제 empty state 문구·레이아웃이 갱신된 `screen-spec-admin.md` A0-c 상태 설명과 일치하는지만 검증하면 된다.

## 3. 변경 범위

### 3.1 라우터 (`admin_router.dart`)

```dart
// 변경 전
if (location == '/login') {
  return ref
      .read(campListProvider)
      .whenOrNull(
        data: (camps) => camps.isEmpty ? '/setup-wizard' : '/camps',
      );
}

// 변경 후 — 캠프 개수 분기 제거, 로그인 성공 시 항상 캠프 목록
if (location == '/login') {
  return '/camps';
}
```

`campListProvider`를 더 이상 이 분기에서 읽지 않게 되므로, `_AdminRouterRefresh`(line 126-134)의 `campListProvider` 리스너가 이 redirect 목적만으로 존재했는지 확인 — 다른 곳(예: PENDING/ACTIVE/ENDED 상태 분기)에서도 캠프 목록 변경에 반응해야 하면 리스너는 유지.

### 3.2 문서 (완료됨)

- `docs/front/screen-spec-admin.md` A0 "후속 분기" — 캠프 개수 무관 항상 A0-c로 이동하도록 수정 완료.
- `docs/front/screen-spec-admin.md` A0-c "상태" — empty state가 이제 정상 경로임을 반영 완료.
- `docs/front/scenarios.md` Feature 2-d, 2-g — 로그인 시점 자동 분기 시나리오를 "항상 캠프 목록으로 이동" 시나리오로 교체 완료.

### 3.3 영향받지 않는 부분

- A0-b 초기 설정 마법사 자체의 내용(3단계 스텝)은 변경 없음 — 진입 경로만 "+ 새 캠프 시작" 버튼으로 단일화.
- PENDING/ACTIVE/ENDED 캠프 선택 후의 사이드바 모드 분기(`_redirect` line 111-123)는 이번 변경과 무관, 그대로 유지.

## 4. 검증 체크리스트 (구현 시)

- [ ] 캠프 0개 상태에서 로그인 → `/camps`로 이동하고 empty state가 보인다(자동으로 `/setup-wizard`로 가지 않는다).
- [ ] 캠프 1개 이상 상태에서 로그인 → `/camps`로 이동한다(기존과 동일, 회귀 없음).
- [ ] 캠프 목록에서 "+ 새 캠프 시작" 클릭 → `/setup-wizard`로 정상 진입한다(기존 동작 유지).
- [ ] 마법사 1단계에서 "← 캠프 목록"으로 취소 → `/camps`로 복귀한다(기존 동작 유지).
- [ ] `admin_router_test.dart` 또는 동등한 라우터 테스트가 있다면, "캠프 0개 로그인 시 `/setup-wizard`로 리다이렉트" 단정을 "캠프 0개 로그인 시 `/camps`로 리다이렉트"로 갱신한다.
