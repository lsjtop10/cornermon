# 앱 전역 네트워크 설정 환경변수 주입 계획

## 1. 개요 및 변경 사유

현재 `api_client.dart`와 `sse_client.dart`에 API base URL, HTTP 커넥트/리시브 타임아웃, SSE 하트비트 워치독 타임아웃이 리터럴로 하드코딩되어 있어, 스테이징/프로덕션 빌드나 로컬 환경마다 다른 백엔드 주소·타임아웃 값을 쓰려면 소스 수정 후 재빌드해야 합니다.

Flutter는 런타임에 OS 환경변수를 직접 읽을 수 없으므로(모바일/웹 샌드박스 제약), `--dart-define` / `--dart-define-from-file`로 **컴파일 타임에 주입**하는 표준 방식을 채택합니다. 새 의존성 추가 없이 Dart SDK 내장 `String.fromEnvironment`/`int.fromEnvironment`만 사용합니다.

**대상 범위**: 사용자 확인 결과, `api_client.dart`의 `baseUrl` + `connectTimeout`/`receiveTimeout`, `sse_client.dart`의 `heartbeatTimeout` 전부를 포함합니다(같은 파일·같은 성격의 네트워크 설정 하드코딩이므로 함께 정리).

기본값(주입 안 했을 때)은 기존 하드코딩 값과 동일하게 유지하여, `flutter run`/`flutter test`를 인자 없이 실행하는 기존 워크플로가 깨지지 않도록 합니다.

## 3. 객체 및 인터페이스 정의

### 3.1 `AppEnv` 신규 도입 (`lib/shared/config/app_env.dart`, 신규 파일)

Flutter/Dio 등 외부 의존성이 없는 순수 Dart 상수 클래스. `import_analysis_options.yaml`의 `shared_no_apps` 규칙(admin/facilitator import 금지)만 자연히 지키면 되고, 새 lint 규칙 추가는 불필요합니다.

```dart
/// 앱 전역 네트워크 설정. --dart-define(-from-file)로 컴파일 타임에 주입되며,
/// 값을 넘기지 않으면 로컬 개발 기본값으로 동작한다.
class AppEnv {
  const AppEnv._();

  static const String apiBaseUrl = String.fromEnvironment(
    'API_BASE_URL',
    defaultValue: 'http://localhost/api/v1',
  );

  static const int apiConnectTimeoutMs = int.fromEnvironment(
    'API_CONNECT_TIMEOUT_MS',
    defaultValue: 5000,
  );

  static const int apiReceiveTimeoutMs = int.fromEnvironment(
    'API_RECEIVE_TIMEOUT_MS',
    defaultValue: 5000,
  );

  static const int sseHeartbeatTimeoutSeconds = int.fromEnvironment(
    'SSE_HEARTBEAT_TIMEOUT_SECONDS',
    defaultValue: 40,
  );
}
```

### 3.2 `api_client.dart` 수정 (기존 파일 확장)

- 책임: 기존과 동일(Dio 인스턴스 생성 + AuthInterceptor 부착). `BaseOptions`의 리터럴 3곳만 `AppEnv` 참조로 교체.

```dart
Dio apiClient(Ref ref) {
  final dio = Dio(
    BaseOptions(
      baseUrl: AppEnv.apiBaseUrl,
      connectTimeout: Duration(milliseconds: AppEnv.apiConnectTimeoutMs),
      receiveTimeout: Duration(milliseconds: AppEnv.apiReceiveTimeoutMs),
    ),
  );
  ...
}
```

### 3.3 `sse_client.dart` 수정 (기존 파일 확장)

- 책임: 기존과 동일. 생성자 기본 파라미터만 `AppEnv` 참조로 교체 (인스턴스별로 여전히 override 가능해야 하므로 named parameter 구조는 유지).

```dart
SseClient(
  this._dio, {
  this.heartbeatTimeout = const Duration(seconds: AppEnv.sseHeartbeatTimeoutSeconds),
});
```

> `sseClientProvider`(파일 하단 `@riverpod` 함수)는 수정 불필요 — 생성자 기본값을 그대로 상속받음.

### 3.4 `env/dev.json` 신규 파일 (frontend 루트, `--dart-define-from-file` 예시)

기존 하드코딩 값과 동일한 내용을 파일로 옮겨, 로컬 개발자가 값을 바꾸고 싶을 때 소스 수정 없이 이 파일만 복사/수정해 쓸 수 있게 합니다. 시크릿이 아니므로 커밋 대상입니다.

```json
{
  "API_BASE_URL": "http://localhost/api/v1",
  "API_CONNECT_TIMEOUT_MS": "5000",
  "API_RECEIVE_TIMEOUT_MS": "5000",
  "SSE_HEARTBEAT_TIMEOUT_SECONDS": "40"
}
```

## 6. 후속 과제 (Follow-up Tasks)

이번 계획의 범위는 `AppEnv` 도입과 기본 배선 교체까지입니다. 아래 두 가지는 이번 Phase A~D 완료 후 별도로 진행할 후속 과제로, 이번 계획의 검증 체크리스트에는 포함하지 않습니다.

### 6.1 실행 명령어 문서화 확장

Phase D-2(README 실행 방법 섹션)는 `--dart-define-from-file`/`--dart-define` 기본 예시만 다룹니다. 후속으로 앱이 admin/facilitator 두 진입점(`lib/main_admin.dart`, `lib/main_facilitator.dart`)과 로컬/스테이징/프로덕션 등 여러 환경 조합으로 실행될 것을 감안해, 조합별 실행 명령어를 표로 정리합니다.

| 항목 | 설명 |
| --- | --- |
| 대상 파일 | `frontend/README.md` (Phase D-2에서 만든 섹션 확장) 또는 `frontend/docs/run-commands.md` (신규, 표가 길어질 경우) |
| 내용 | 진입점(`main_admin.dart` / `main_facilitator.dart`) × 환경(local/staging/prod) × 플랫폼(web/ios/android) 조합별 `flutter run`/`flutter build` 명령어 매트릭스 |
| 전제 조건 | 스테이징/프로덕션용 `env/*.json` 값(실제 배포 API 주소 등)이 먼저 확정되어야 함 — 현재는 로컬 기본값만 존재 |
| 비고 | CI 파이프라인이 아직 없으므로(§4 Phase D 하단 비고 참조), 이 문서화는 로컬 수동 실행 기준으로 작성하고 CI 스크립트화는 파이프라인 구축 시점에 다시 다룸 |

### 6.2 개발 가이드라인 추가

새 네트워크/전역 설정값을 추가하려는 개발자가 `AppEnv`를 우회해 리터럴을 다시 하드코딩하는 회귀를 막기 위해, 가이드라인 문서에 규칙을 명문화합니다.

| 항목 | 설명 |
| --- | --- |
| 대상 파일 | `frontend/README.md`(개발 가이드 섹션) 또는 `workflow/implement.md`(프론트 공용 규칙일 경우) |
| 규칙 초안 | 1) 새 컴파일타임 설정값은 반드시 `lib/shared/config/app_env.dart`의 `AppEnv`에 `String.fromEnvironment`/`int.fromEnvironment` + `defaultValue`로 추가 2) 호출부(`api_client.dart` 등)에 리터럴 재도입 금지 3) 키 네이밍은 `SCREAMING_SNAKE_CASE`, 도메인 접두어 유지(`API_*`, `SSE_*`) 4) `AppEnv`에 값을 추가/변경하면 `env/dev.json`도 함께 갱신 5) 시크릿(토큰, 키 등)은 `--dart-define`/`env/*.json` 대상에서 제외 — 이 경로는 커밋되는 비밀 아닌 설정값 전용 |
| 검증 아이디어 | 코드 리뷰 체크리스트 항목으로 추가하거나, 여유가 되면 `grep -rn "Duration(seconds:\|http://\|https://"` 류의 리터럴 탐지를 PR 템플릿/CI 린트에 추가하는 것도 고려 (이번 계획 범위 밖, 후속 검토) |
