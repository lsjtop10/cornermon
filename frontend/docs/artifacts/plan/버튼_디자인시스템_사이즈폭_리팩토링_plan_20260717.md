# 버튼 디자인 시스템 사이즈/폭 리팩토링

## 배경 / 문제

`AppButton`(`frontend/lib/shared/design_system/widgets/app_button.dart`)을 관리자·진행자 앱이
공유하는데, 두 목업의 실측 CSS가 서로 다르다.

| | 관리자 (`관리자 앱 화면 리뷰 — 코너학습_files/saved_resource.html` `.btn`) | 진행자 (`진행자 앱 화면 리뷰 — 코너학습_files/saved_resource.html` `.btn`) |
|---|---|---|
| height | 34px | 52px |
| border-radius | 9px | 12px |
| font | 12.5px / 700 | 16px / 600 |
| padding-h | 14px | 24px (`.cta-btn`는 pill) |
| 기본 폭 | hug (inline-flex) | **fill (`width:100%`)** |

현재 `AppButton`은 단일 스타일(패딩 24/16, radius 12, `AppTypography.bodyEmphasis` 18px)만 제공해서
진행자 쪽 크기가 관리자에도 그대로 적용되고 있다. 관리자 화면(테이블 액션, 다이얼로그, 툴바)에는 과도하게
크고, 진행자 화면(스마트폰 터치 타깃)에는 폭 지정이 없어 호출부마다 `SizedBox(width: double.infinity)`로
직접 감싸는 중복이 반복된다 (`track_confirm_screen.dart`, `device_pending_screen.dart`, `qr_scan_screen.dart`,
`main_track_screen.dart`, `_main_track_body.dart`, `login_screen.dart` 6곳 확인).

`AppDropdown`도 `height: 36 / radius: 8`로 관리자 `.btn`(34/9)과 미묘하게 어긋나 툴바에서 버튼과 나란히
놓일 때 높이가 안 맞는다.

## 객체 설계

### 신규: `frontend/lib/shared/design_system/tokens/dimensions.dart`

```dart
/// 버튼·드롭다운 등 컨트롤이 공유하는 크기 토큰.
/// compact = 관리자(데스크톱/태블릿), comfortable = 진행자(스마트폰) 실측값.
class AppDimensions {
  AppDimensions._();

  static const double controlHeightCompact = 34.0;
  static const double controlHeightComfortable = 52.0;

  static const double controlRadiusCompact = 9.0;
  static const double controlRadiusComfortable = 12.0;

  static const double iconButtonCompact = 34.0;
  static const double iconButtonComfortable = 38.0;
}
```

### 변경: `app_button.dart`

```dart
enum AppButtonVariant { primary, secondary, destructive, iconOnly }

/// 플랫폼이 아니라 밀도를 나타내는 이름 — compact(고밀도) / comfortable(저밀도 터치).
enum AppButtonSize { compact, comfortable }

/// hug: 글자 길이만큼(기본값) · fill: 부모 폭 100% · fixed: fixedWidth로 고정.
enum AppButtonWidth { hug, fill, fixed }

class AppButton extends StatelessWidget {
  const AppButton({
    required this.variant,
    required this.size,
    required this.label,
    required this.onPressed,
    this.width = AppButtonWidth.hug,
    this.fixedWidth, // width == fixed일 때만 사용, 그 외 assert로 차단
    this.icon,
    this.disabledReason,
    super.key,
  });

  final AppButtonVariant variant;
  final AppButtonSize size;
  final AppButtonWidth width;
  final double? fixedWidth;
  final String label;
  final VoidCallback? onPressed;
  final IconData? icon;
  final String? disabledReason;

  // 책임: size → (height, radius, 폰트, 패딩, 아이콘 정사각 한 변) 매핑
  _ButtonMetrics _metricsFor(AppButtonSize size);

  // 책임: width 전략에 따라 buttonCore를 SizedBox(width: ...)로 감쌀지 결정
  Widget _applyWidth(Widget buttonCore);
}

// private, app_button.dart 내부에만 존재
class _ButtonMetrics {
  const _ButtonMetrics({
    required this.height,
    required this.radius,
    required this.textStyle,
    required this.paddingHorizontal,
    required this.iconSquare,
  });
  final double height;
  final double radius;
  final TextStyle textStyle;
  final double paddingHorizontal;
  final double iconSquare;
}
```

`size` 필드는 `variant`처럼 필수(named required)로 둔다 — 기본값을 두면 관리자 호출부가 여전히
컴포터블 크기로 남아 이번 리팩토링의 목적(관리자/진행자 시각 정렬)이 흐려지기 때문에, 모든 호출부가
명시적으로 크기를 선언하도록 강제한다.

`fixedWidth`는 `width != fixed`일 때 무시되며, `width == fixed`인데 `fixedWidth == null`이면
`assert`로 개발 중 실패시킨다 (런타임 방어 로직 아님, 디버그 빌드 전용 계약 위반 체크).

### 변경: `app_dropdown.dart`

`height: 36` → `AppDimensions.controlHeightCompact`, `borderRadius: 8` → `AppDimensions.controlRadiusCompact`.
`AppDropdown`은 현재 관리자 툴바에서만 쓰이므로 별도 size 파라미터는 추가하지 않는다 (YAGNI).

## 호출부 반영 계획 (Phase)

### Phase A — 토큰 + 위젯 (예상 1.5시간)

| 순서 | 작업 | 파일 |
|---|---|---|
| A-1 | `AppDimensions` 신규 생성 | `frontend/lib/shared/design_system/tokens/dimensions.dart` (신규) |
| A-2 | `AppButton`에 `size`/`width`/`fixedWidth` 추가, `_ButtonMetrics` 매핑 구현, `iconOnly` 정사각 크기도 size 분화 | `frontend/lib/shared/design_system/widgets/app_button.dart` |
| A-3 | `AppDropdown` 높이/라운드를 `AppDimensions.controlHeightCompact` / `controlRadiusCompact`로 교체 | `frontend/lib/shared/design_system/widgets/app_dropdown.dart` |

### Phase B — 관리자 호출부: `size: AppButtonSize.compact` 추가 (예상 1시간)

모두 `width` 기본값(`hug`) 그대로 두되, `login_screen.dart`의 로그인 버튼만 기존
`SizedBox(width: double.infinity)` 래핑을 제거하고 `width: AppButtonWidth.fill`로 대체.

- `dashboard_screen.dart` (2곳)
- `device_manage/_device_registration_row.dart` (2곳)
- `session_manage/_active_sessions_card.dart` (2곳)
- `session_manage/_admin_sessions_card.dart` (1곳)
- `session_manage/_locked_devices_card.dart` (2곳)
- `setup_wizard/setup_wizard_screen.dart` (2곳, 기존 `Visibility`/`Stack` 래핑 유지)
- `setup_wizard/steps/corner_track_step.dart` (1곳)
- `start_camp/start_camp_button.dart` (2곳)
- `track_bulk_manage/track_bulk_manage_screen.dart` (2곳)
- `track_direct/_chat_thread_pane.dart` (1곳, iconOnly)
- `login/login_screen.dart` (1곳, **`width: fill`로 변경 + `SizedBox` 래퍼 제거**)

### Phase C — 진행자 호출부: `size: AppButtonSize.comfortable` 추가 + `fill` 래퍼 제거 (예상 1시간)

기존 `SizedBox(width: double.infinity, child: AppButton(...))` 패턴을
`AppButton(..., size: comfortable, width: AppButtonWidth.fill)`로 대체하고 바깥 `SizedBox` 제거.

- `pin_login/pin_login_screen.dart` (1곳, hug 유지 — 보조 링크성 버튼)
- `device_pending/device_pending_screen.dart` (2곳, **fill**)
- `track_direct/track_direct_screen.dart` (1곳, iconOnly)
- `track_confirm/track_confirm_screen.dart` (2곳, **fill**)
- `main_track/_main_track_body.dart` (1곳, **fill**)
- `main_track/main_track_screen.dart` (1곳, **fill**)
- `main_track/_main_track_header.dart` (2곳, iconOnly)
- `qr_scan/qr_scan_screen.dart` (1곳, **fill**)
- `widgets/double_tap_confirm_button.dart` (1곳, hug 유지 — armed 상태 커스텀 스타일과 크기 맞춰 `AppDimensions.controlHeightComfortable`/`controlRadiusComfortable` 참조하도록 함께 정리)

### Phase D — 공유 위젯 (예상 30분)

- `confirm_modal.dart`: `AppButtonVariant.destructive` 버튼에 `size: AppButtonSize.comfortable` 지정
  (현재 이 모달은 진행자·관리자 양쪽에서 쓰이는데, 실사용처가 모두 진행자 쪽 확인 다이얼로그라 comfortable로
  통일 — 실제 관리자 쪽 호출부가 있는지 최종 확인 후 필요 시 파라미터화 여부 재검토).
- `empty_state.dart`: 호출부 확인 후 해당 플랫폼에 맞는 size 지정.

### Phase E — 테스트 갱신 (예상 30분)

`size`가 필수 파라미터가 되므로 기존 위젯 테스트에서 `AppButton(...)` 생성 시 컴파일 에러가 나는 지점을
모두 찾아 `size:` 추가.

- `test/admin/features/dashboard/dashboard_screen_test.dart`
- `test/admin/features/start_camp/start_camp_button_test.dart`
- `test/admin/router/admin_router_test.dart`
- 그 외 `flutter analyze`로 드러나는 모든 컴파일 에러 지점

## 구현 결과 (2026-07-17, 워크트리 `worktree-button-design-system-refactor`)

이 워크트리는 `implement.md` 규칙("특별한 요청이 없으면 main의 최신 변경사항에서 분기")에 따라
**origin/main**에서 분기했다. 계획 수립 시 참고했던 관리자 화면 호출부(`dashboard_screen.dart`,
`start_camp_button.dart`, `track_bulk_manage_screen.dart`, `confirm_modal.dart`, `app_dropdown.dart`)는
아직 커밋되지 않은 별도 작업(`fix/admin-design-system-alignment` 브랜치의 워킹카피)에만 존재하고
main에는 없어 이번 스코프에서 제외했다. 실제로 반영한 범위는 다음과 같다.

- **Phase A**: `AppDimensions` 신규 생성, `AppButton`에 `size`/`width`/`fixedWidth` 추가 완료.
  `app_dropdown.dart`는 main에 존재하지 않아 A-3은 수행하지 않음 — 해당 브랜치가 머지된 뒤 별도로 정렬 필요.
- **Phase B**: main에 실제로 존재하는 관리자 `AppButton` 호출부 8곳만 `size: compact` 반영
  (`_device_registration_row.dart`, `_active_sessions_card.dart`, `_admin_sessions_card.dart`,
  `_locked_devices_card.dart`, `setup_wizard_screen.dart`, `corner_track_step.dart`,
  `_chat_thread_pane.dart`, `login_screen.dart` — 로그인 버튼은 `width: fill`도 함께 적용).
- **Phase C**: 계획대로 진행자 9곳 전부 `size: comfortable` 반영, `SizedBox(width: double.infinity)` 래퍼는
  `width: AppButtonWidth.fill`로 대체하고 제거. `double_tap_confirm_button.dart`의 armed 상태 커스텀 스타일도
  `AppDimensions.controlHeightComfortable`/`controlRadiusComfortable`로 정리.
- **Phase D**: `confirm_modal.dart`는 main에서 `AppButton`을 쓰지 않아(ElevatedButton 직접 사용) 스코프 밖.
  `empty_state.dart`는 `actionButtonSize`(기본값 `compact`) 파라미터를 추가해 대응 — 현재 액션 버튼이 있는
  호출부 5곳이 모두 관리자 화면이라 기본값만으로 충분했고, 호출부 자체는 수정하지 않았다.
- **Phase E**: `AppButton`/`EmptyState`를 직접 생성하는 테스트 파일이 없어 컴파일 에러가 발생하지 않았다
  (화면을 통째로 pump하는 통합 테스트뿐). 별도 테스트 수정 불필요, `flutter test` 142개 전부 통과.
