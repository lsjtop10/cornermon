### 종료 조건

- 기기 신뢰 토큰, 등록 ID, 로컬 신뢰 상태, 저장된 트랙 세션이 모두 제거된다.
- 라우터는 `DeviceTrustStatus.none`을 받아 `/device-pending`으로 이동한다.
- SSE는 즉시성 신호일 뿐이고, `DEVICE_NOT_TRUSTED` 응답만으로도 동일한 결과에 도달한다.
- 기존 API 계약과 생성 코드 변경은 없다.

## 2. 설계

신뢰 해제의 단일 책임은 `DeviceTrust` notifier에 둔다. 기존의 REVOKED 폴링 처리와 SSE·PIN 실패·사용자 액션이 모두 이 메서드를 호출해 저장소 키와 provider 상태가 어긋나지 않게 한다.

```dart
@riverpod
class DeviceTrust extends _$DeviceTrust {
  /// 등록 코드 입력을 다시 받을 수 있도록 로컬 신뢰 정보를 원자적으로 초기화한다.
  Future<void> clearRegistration();
}

Future<void> _handleCampEnded() async {
  await ref.read(deviceTrustProvider.notifier).clearRegistration();
  ref.read(trackSessionProvider.notifier)
      .handleTermination(TrackSessionTerminationReason.campEnded);
}
```

- `clearRegistration()`은 폴링을 중지하고 `device_trust_token`, `device_trust_status`, `device_trust_registration_id`를 지우며 `deviceTrustTokenProvider`를 invalidate한다.
- 상태를 `none`으로 전이하면 기존 `facilitator_router.dart`의 최우선 가드가 모든 진행자 화면에서 `/device-pending`으로 보낸다.
- 수동 초기화는 서버 기기 등록을 임의로 REVOKED로 바꾸지 않는다. 서버의 신뢰 철회는 캠프 종료/관리자 회수의 권한으로 유지하고, 이 UI는 해당 기기의 **로컬 등록 정보만** 지운다.

## 4. API 계약 영향

기존 `camp_ended` SSE와 `DEVICE_NOT_TRUSTED` 에러 코드만 사용한다. 새 REST 엔드포인트나 OpenAPI 변경은 없으므로 `api/swagger.yaml` 및 `lib/shared/api/gen`은 수정하지 않는다.
