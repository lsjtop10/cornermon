## 기존 코드 이해

- `frontend/lib/admin/features/group_detail/group_detail_screen.dart`는 현재 `itinerary`의 모든 코너를 동일한 `Chip`으로 나열해 완료 비율과 현재 위치를 빠르게 파악하기 어렵다.
- `frontend/lib/admin/entities/group_ext.dart`는 완료 수와 전체 수를 이미 계산한다. 진행률 계산만 보완하고, 코너별 상태는 순회표 원본에서 그대로 표시한다.
- API의 `CornerProgressResponse`는 `NOT_VISITED`, `IN_PROGRESS`, `COMPLETED` 상태와 코너 이름을 제공하므로 계약 변경은 필요 없다.

## 객체 설계

extension AdminGroupX on api.Group {

책임: API 순회표의 계산 가능한 표시 상태를 화면에 제공한다. 빈 순회표에서는 진행률을 `0`으로 안전하게 반환한다.
